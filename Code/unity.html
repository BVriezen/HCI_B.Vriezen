Unity & Madmapper

Unity is een programma dat gebruikt wordt om games en andere simulaties in te maken. Binnen de tweede workshop voor het vak HCI was het de bedoeling om een aantal opdrachten binnen Unity te maken om zo te leren om met het programma over weg te kunnen.

Hier heb ik een scene gemaakt met daarbinnen een plane, een plane is een soort ondergrond waar dingen op geplaats kunnen worden. Vervolgens heb ik doormiddel van de functie Raise Terrain de plane op een aantal plekken verhoogd. Hierdoor ontstaan er bergen en andere puntige vormen. Vervolgens heb ik een asset gedownload waardoor ik instaat ben om binnen de omgeving te lopen doormiddel van een camera.

Een asset kan gezien worden als een extra voorwerp/functionaliteit die je toegevoegd binnen het Unity bestand.

Vervolgens heb ik in de scene een aantal 3d objecten toegevoegd, zoals is te zien zweven sommige objecten boven de grond. Dit komt doordat ik ze niet hellemaal tegen de plane aangedrukt zijn.

In het derde filmpje is te zien dat de buis als eigenschap heeft dat deze solide is, maar wel verplaats kan worden als er tegenaan gelopen wordt. Dit is bij de andere voorwerpen niet, bij deze voorwerpen kan je erdoorheen lopen. Ik heb de buis via de functie Physics zo aangepast dat de buis solide is en dat de buis verplaatst kan worden.

De volgende stap bestond uit het toevoegen van een asset uit de asset store van Unity, een soort digitale winkel waar je voorwerpen en objecten kan toevoegen en kopen. Hierbij ik gekozen voor een gratis dragonasset die ik toegevoegd heb binnen mijn scene. Vervolgens heb ik een trigger rondom de dragonasset gemaakt, dit wil zeggen dat er een interactie plaats vindt met de asset als een bepaalde actie uitgevoerd wordt door de speler. Ook heb ik een radius/cirkel toegevoegd aan de draak, als iemand binnen deze cirkel stapt gebeurd er iets. In dit geval is dat dat er een vlam voor de kop van de draak verschijnt, deze verdwijnt ook weer als iemand uit de cirkel stapt.

De vlam is hierbij een aparte asset die ik heb gekoppeld aan de draak via een script.

Voor de laatste opdracht moest er nog een extra trigger/ interactie worden toegevoegd aan het spel. Ik heb besloten om een tekst in het spel toe te voegen zodra de speler vlakbij de draak staat. Dit is gedaan doormiddel van het plaatsen van een onzichtbaar object dat een trigger heeft zodra de speler erdoorheen loopt. Het gevolg van de trigger is in dit geval een tekst die weergegeven wordt. De code voor de trigger heb ik van een filmpje vandaan dat op YouTube staat.

Ik weet niet zozeer of mijn spel echt een doel heeft, het is vooral om te laten zien hoe dingen binnen Unity werken. Wat misschien wel leuk is, is dat de tekst verteld dat de draak vriendelijk is, maar als je te dichtbij komt gaat hij vuur ademen. Je kunt dit zien als een soort grapje.
